Background

Introducing the background of the project

MathRacing is a collaborative project between UCL and Morgan Stanley. This project will aim towards enhancing numeracy skills in primary school children through gamification. A number of games and environments already exist that help learning in both literacy and numeracy.

Implementing the concept of gamification, we employed racing game elements to combine with the math questions so that users can engage in and improving learning interest.

Gamification

Improving engagement of users

What is Gamification?

Gamification is the process of applying game playing techniques to enhance engagement and user experience. It can be particularly useful in enhancing learning as students interact with simulated real world artefacts in reference to the concepts they learn in their conventional study. Gamification aids learning through:

1.Virtual interaction with simulated real world environments,
2.Constant evaluation of their actions leading to instant reward and loss ,
3.Scenario adaptation based on skills and knowledge

A solid example

Math Invaders, which is an adaptation of the famous console game Space Invaders. In this game, numbered aliens descend and the defender can only shoot the one whose number matches the answer of an arithmetic problem on the screen allowing players to practice mental arithmetic.

And the ultimate goal of this project is, by using gamification as a method, we are going to build a tool for primary school children, teachers and parents to help improve numeracy skills and also understand practical applications of these skills.

Technologies

Technologies and researches

Game engine and template


HTML5, Php, Javascript

HTML5 is a markup language for World Wide Web. It is the fifth and latest version of HTML standard. It specified scripting API that can be used in JavaScript. HTML also supports mySQL database engine, these are why we use HTML5 for login page and the game console.

PHP is a server-side language for web development. It can be embedded into HTML5. We use it for structuring the web content, connecting database with website and submiting data.

JavaScript is dynamic, prototype-based language. It has several advantages(source: http://www.jscripters.com/javascript-advantages-and-disadvantages):

1. Javascript is executed on the client side.
This means that the code is executed on the user's processor instead of the web server thus saving bandwidth and strain on the web server.

2.Javascript is a relatively easy language The Javascript language is relatively easy to learn and comprises of syntax that is close to English. It uses the DOM model that provides plenty of prewritten functionality to the various objects on pages making it a breeze to develop a script to solve a custom purpose.

3. Javascript is relatively fast to the end user
As the code is executed on the user's computer, results and processing is completed almost instantly depending on the task (tasks in javascript on web pages are usually simple so as to prevent being a memory hog) as it does not need to be processed in the site's web server and sent back to the user consuming local as well as server bandwidth.

4.Extended functionality to web pages
Third party add-ons like Greasemonkey enable Javascript developers to write snippets of Javascript which can execute on desired web pages to extend its functionality. If you use a website and require a certain feature to be included, you can write it yourself and use an add-on like Greasemonkey to implement it on the web page.

Importantly, one is our client requires the game should be run on all platforms. So HTML5, PHP and JavaScript are perfect since it is compatible on all mainstream platforms

Phaser

We want to work with a Javascript framework that has OOP structure, and this is also the first time we are working on a web game, so we want to choose a language that offers maximum result with minimum efforts. So we decide to go with Phaser for the following reason.
1. Phaser can draw and animate sprites
2. Phaser allow user Input (keyboard,mouse and touchscreen)
3. Phaser can draw Fonts
4. Phaser can handle physics
5. Phaser can play sound
6. Support on most modern devices
7. Well-written documentation

Plugin

JQuery: a fast, small, and feature-rich JavaScript library. This is used in this project to handle AJAX communication between server and game console.

Database

mySQL: mySQL is an in-process library that implements a self-contained, serverless, zero-configuration, transactional SQL database engine. It reads and writes directly to ordinary disk files. A complete SQL database with multiple tables, indices, triggers, and views, is contained in a single disk file. Compared to mySQL it is a lightweight database engine that includes all the requirements we need. Another reason we choose it is that we do not have much cucurrency works in our game. So sqlite is an ideal free-to-use engine for our game.

phpMyAdmin: we use phpMyAdmin to visualise project database. It is extremely handy to import/export databse as sql file. It also helps to check the effectivity of submitting data.
