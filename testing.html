2.Functionality Testing

Test for - all the stages in game, database connection, game console.
See more at Functional Requirement in about the game.

Test for - all the links in web pages, database connection, forms used in the web pages for submitting or getting information from user.
Check all the links:
● Test the outgoing links for credibility.
● Test all internal links.
● Test to check if there is any page that doesn’t link to others.

Test forms in all pages:
Forms are used to registration and login for the game. We carry this test by inputting the data ourselves to see if we get the correct output.
● Password should be at least 6 characters
● Password and confirm password should be the same
● Wrong inputs to the fields in the forms.

Database testing:
● Data consistency is very important in web application especially web game. Check for data integrity and errors while you edit, delete, modify the forms or do any DB related functionality.
● Verify data integrity and errors while you edit, delete modify when you do anything DB related (creating new user, create new game session)
● Check if SQL query is executed correctly and the result of the database is returned correctly.

3. Usability Testing:

Test for navigation:
The user should be able to move through the whole webpage and game state swiftly. There should not be any dead-end.

Content checking:
Content should be easy to understand. The manual should be informative and contains all the information the user need. Check if there is any spelling error and try to not use too dark or too bright colour.

4. Interface Testing:

The main interfaces are:
Web server and application server interface
Application server and Database server interface.
Check if the interaction between front-end and back-end is handled correctly. Try to find if any error is not handled by the system.

5. Compatibility Testing:

Compatibility of your web site is very important testing aspect. See which compatibility test to be executed:

Browser compatibility:
The game will work on most modern browsers, which are newer than Internet Explorer 7.0.

Mobile browsing:
The game will run on all touch devices

6. Performance testing:

Web performance testing will include the below 2 tests:

Web Load Testing
Test the performance on different Internet connection speed. How many requests can the system handle simultaneously? How large the input data from user the system can accept?

Web Stress Testing
Under which circumstance the system will crash? How the system recovers from crash?

7. Security Testing:

Following are some test cases for web security testing:
● Test by pasting internal URL directly into browser address bar without session. Internal pages should not open.
● Try some invalid inputs in input fields like login username, password, input text boxes. Check the system reaction on all invalid inputs.
● Web directories or files should not be accessible directly unless given download option.
● All transactions, error messages, security breach attempts should get logged in log files somewhere on web server.

8. Algorithm Testing:

● The algorithm should not created 2 same questions in 1 session
● Algorithm should be able to create a new question fast
● Avoid of having question with same first number
